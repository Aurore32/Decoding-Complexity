Algorithms for Bowtie Problem

  • Problem: so far, we've observed the following two problems in the integrated ownership model -
    1. SCCs lead to disproportionately high integrated values due to self-loops
    2. Root nodes lead to accumulation of flow
  • The first problem was solved with the correction operator DDD
  • Two adjustments can be made to the model to correct the second problem

Analytical solution to the bowtie problem

  • Definition. Define the bowtie-corrected integrated ownership matrix as Wˉ=W(W^+D)\bar{W}=W(\hat{W}+D)Wˉ=W(W^+D), where W^\hat{W}W^ denotes the self-loop-corrected integrated ownership W^=DW~\hat{W}=D\tilde{W}W^=DW~.
  • Derivation. First, observe that because we defined the self-loop-corrected network value as vnet^=Dvnet\hat{v^{net}}=Dv^{net}vnet^=Dvnet, we have the corrected integrated value

    v^net=Dvnet=Dνint+Dv=DW~v+Dv=W^v+Dv\begin{aligned} \hat{v}^{net} &= Dv^{net} \\ &=D\nu^{int}+Dv \\ &=D\tilde{W}v+Dv \\ &=\hat{W}v+Dv \end{aligned}v^net​=Dvnet=Dνint+Dv=DW~v+Dv=W^v+Dv​

Algorithmic implementation

  • The main issue with the above corrections is computational speed: matrix inversion has complexity O(n3)\mathcal{O}(n^3)O(n3), which can become infeasible when dealing with very large matrices (e.g. a network of all firms in a country)
  • An algorithm involving the following steps can be implemented to calculate network value for any given node iii, through which integrated value can be easily found:
  1. Breadth-first search (BFS) is implemented on node iii, returning a set of nodes that are reachable from node iii (in however many steps). Call this set Si\mathcal{S_i}Si​.
  2. S\mathcal{S}S represents a subnetwork of the original network; all nodes in this network are connected, directly or indirectly, to iii. Without loss of generality, relabel the node iii to 111, and the other nodes 222, 333, etc. Call the adjacency matrix derived from this subnetwork B(i)B(i)B(i).
  3. Remove all the incoming links to iii such that now, no firm owns any shares in iii. B(i)B(i)B(i) now looks like this:
    B(i)=[0d⃗0⃗Bsub]B(i)=\begin{bmatrix} 0 & \vec{d} \\ \vec{0} & B^{sub} \end{bmatrix}B(i)=[00​dBsub​]
    where d⃗\vec{d}d is the row matrix representing the ownership of firm 1 in other firms (its outgoing links), BsubB^{sub}Bsub is a matrix representing the adjacency network of the subnetwork aside from node 111, and 0⃗\vec{0}0 is the zero vector. The first column of this adjacency matrix is zero, because no firm owns shares in node 111.
  4. Calculate the network value of the node iii: by definition of network value, vnet=W(I−W)−1vv^{net} = W(I-W)^{-1}vvnet=W(I−W)−1v where vvv is the intrinsic value vector and WWW is the adjacency matrix. Using W=B(i)W = B(i)W=B(i) and v=vsubv = v^{sub}v=vsub yields the network value.

(Theorem.) The operation of removing all incoming links to iii for the subnetwork represented by B(i)B(i)B(i) is mathematically equivalent to performing the self-loop correction on B(i)B(i)B(i): B(i)^=DB(i)\hat{B(i)}=DB(i)B(i)^​=DB(i).

(Proof.) Recall that the corrected integrated ownership matrix W^\hat{W}W^ was defined in scalar form as:

W^ij=Wij+∑k≠iW^ikWkj\hat{W}_{ij}=W_{ij}+\sum_{k\neq i} \hat{W}_{ik}W_{kj}W^ij​=Wij​+k=i∑​W^ik​Wkj​

As the subnetwork with ownership matrix BBB instead of WWW preserves all the outgoing connections WikW_{ik}Wik​ of iii but removes all the incoming connections WkiW_{ki}Wki​, B^ij\hat{B}_{ij}B^ij​ can no longer contain any self-loops; thus, B^ij\hat{B}_{ij}B^ij​ is equal to W^ij\hat{W}_{ij}W^ij​ with W^ii\hat{W}_{ii}W^ii​, the self-loop coomponent, removed. This leads to

B^=W^−diag(W^)\hat{B}=\hat{W}-\text{diag}(\hat{W})B^=W^−diag(W^)

in matrix form, and with

D=I−diag(W^)D = I - \text{diag}(\hat{W})D=I−diag(W^)

we have B^=D+W^−I\hat{B} = D + \hat{W} - IB^=D+W^−I. Thus B^v+v\hat{B}v + vB^v+v, the network value of the subnetwork by definition, is equal to

Dv+W^v−v+v=W^v+Dv=v^netDv + \hat{W}v - v + v = \hat{W}v + Dv = \hat{v}^{net}Dv+W^v−v+v=W^v+Dv=v^net

which is the self-loop corrected network value for the original network.

  • Correcting for bowtie structures.

  • Definition. Three subsets of a bowtie-structured network. A bowtie-structured network can be divided roughly into the following three subsets:

  1. O\mathcal{O}O, the OUT subset, containing the labels of all nodes that are leaf nodes - that is, with only incoming connections and no outgoing connections.
  2. I\mathcal{I}I, the IN subset, containing all root nodes - nodes that have outgoing connections and no incoming connections. As noted, these nodes are where integrated value aggregates erroneously.
  3. S\mathcal{S}S, the SCC. We have all of O,I,S⊂Si\mathcal{O}, \mathcal{I}, \mathcal{S} \subset \mathcal{S_i}O,I,S⊂Si​, the subnetwork formed from breadth-first search on node iii. We also have O∩I=∅{\mathcal{O} \cap \mathcal{I}} = \emptysetO∩I=∅.
  • We apply the algorithm detailed above to calculate vnetv^{net}vnet in the following order:
  1. On O\mathcal{O}O, all the nodes in the OUT subset.
  2. On SOS_OSO​, all the nodes in the SCC that are directly connected to O\mathcal{O}O. Calculate their network values by applying the algorithm above, or simply by vnet(s)=∑oWsovnet(o)v^{net}(s) = \sum_{o} W_{so}v^{net}(o)vnet(s)=∑o​Wso​vnet(o) for s∈SOs\in S_Os∈SO​ and o∈Oo \in \mathcal{O}o∈O connected to node sss: the network value of all the OUT nodes connected to sss, weighted by the weight of the links WsoW_{so}Wso​. This network value is added to the original intrinsic values of these nodes: vs→vnet(s)+vsv_s \to v^{net}(s) + v_svs​→vnet(s)+vs​.
  3. On the SCC itself S\mathcal{S}S, not including any nodes in O\mathcal{O}O or I\mathcal{I}I. Apply the algorithm above to find their network values vnetv^{net}vnet, taking into account that nodes within the SCC connected to O\mathcal{O}O had their intrinsic values modified as per above.
  4. On SI⊂IS_I \subset \mathcal{I}SI​⊂I: the subset of the IN set that is directly connected to the SCC S\mathcal{S}S. This is the subset that will be adjusted to correct for accumulation of flow. We directly set for all s∈SIs \in S_Is∈SI​: vnet(s)=∑iWsivnet(i)v^{net}(s) = \sum_{i} W_{si}v^{net}(i)vnet(s)=∑i​Wsi​vnet(i), ensuring that the network value of these IN nodes will not be the accumulated value of every node in the SCC, but rather, only the weighted sum of the values of the nodes they are connected to.
  5. On I\mathcal{I}I, taking note to add the network values calculated in the previous step to the intrinsic value of the nodes.
